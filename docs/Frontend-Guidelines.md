# Frontend Guidelines

## ESLint/TS/Prettier

-   Use the Airbnb JavaScript Style Guide.
-   Use TypeScript for all new components.
-   Use Prettier for code formatting.
-   Import order:
    1.  Built-in modules
    2.  Third-party modules
    3.  Local modules

## CSS/Tailwind Tokens

-   Use CSS/Tailwind tokens for theming.
-   Use consistent naming conventions for theme variables.

## RBAC/Tenancy Hooks

-   Use RBAC/tenancy hooks for access control.
-   Use consistent naming conventions for RBAC/tenancy variables.

## Data Contract

-   Define a clear data contract for all API requests and responses.
-   Use TypeScript interfaces to define the data contract.

## PR Message/Commit Convention

-   Use the following format for PR messages:
    `feat(component): Add new component`
-   Use the following format for commit messages:
    `feat(component): Add new component`
